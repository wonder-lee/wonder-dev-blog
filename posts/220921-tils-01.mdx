## 내용

### Git은 file의 대소문자 변경사항을 추적하지 않는다.

작업 중 project의 file 하나가 convention과 다르게 첫글자가 대문자 형식으로 되어있는 file있어서 camelCase로 수정하였다. 그리고 작업과 PR이 모두 완료되어 배포를 하려는 순간 error가 발생하였고, error 내용에는 제가 수정한 file 명이 변경되지 않아있었다.

검색 중 `Git은 file의 대소문자 변경사항을 추적하지 않는다.` 는 점을 알게 되었다.

### 해결방법

1. folder, file 대소문자 변경 추척하도록 설정

   ```tsx
   git config core.ignorecase false
   ```

2. 변경사항 추적한 점을 확인 후 `push`
3. 원격에서 확인하면 기존 file, 변경 file 2개가 존재하므로, 기존 file 삭제합니다.

이렇게 하면 정상적으로 변동이되지만, 저의 경우에는 local git에서 계속해서 `untracked working tree files overwritten` error가 발생하여 할 작업이 많아 우선 무식하게…새로 clone을 받아 local error를 회피해버렸다..
